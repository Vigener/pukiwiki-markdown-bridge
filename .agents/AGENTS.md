# エージェントへの行動ルール (Agent Behavioral Rules)

<RULE[test_before_commit]>
## 自動テストの徹底
コードを変更し、Gitでコミットおよびプッシュする際は、**必ず自律的に事前のローカルテストを実行し、エラーが出ないことを確認する**こと。
ユーザーに「バグがないか手動で動作確認してもらう」という前提でコードをプッシュしてはならない。

1. **コンバーターロジックの変更**: `src/converter.ts` などのロジックを変更した場合は、必ず `npx tsx test_roundtrip.ts` 等のテストスクリプトを実行し、文法エラーやロジックエラーがないことを確認する。
2. **ビルドの確認**: `npm run build` を実行して、ビルドが通るか確認する。
3. 検証が完了し、問題がないことを確認してから初めて `git commit` および `git push` を行う。
</RULE[test_before_commit]>
