# PukiWiki Markdown Bridge

PukiWikiの編集を、よりモダンなMarkdownエディタ（[EasyMDE](https://github.com/Ionaru/easy-markdown-editor)）で行うためのChrome拡張機能です。

## 特徴
- PukiWikiの編集画面（テキストエリア）の内容を自動的にMarkdownに変換してポップアップエディタに表示
- Markdownでの編集後、ワンクリック（またはショートカットキー）でPukiWiki構文に再変換し、元のテキストエリアへ反映
- 見出し、太字、斜体、リスト、リンクなど一般的なMarkdown構文をサポート
- 誤ってタブを閉じてしまった際などに復元できる **ドラフト自動保存機能**
- 反映前やドラフト復元前に変更点を確認できる **Diff（差分）確認機能**
- PukiWiki非互換の記述（直接のHTMLテーブル等）を事前検知する **バリデーション警告機能**
- PukiWikiの仕様に合わせて自動調整（簡略化）される箇所を事前確認できる **自動調整の確認機能**
- テキストエリアが見つからない閲覧画面でも、ワンクリックで編集画面へ遷移できる **[編集画面へ移動] ボタン機能**
- 誤操作防止のため、指定したURL（自身のディレクトリ等）でのみ拡張機能が動作するよう安全に制限可能

## インストール方法（開発・ローカル環境）
1. 本リポジトリをクローンまたはダウンロードします。
2. Node.js環境で依存パッケージをインストールし、ビルドします。
   ```bash
   npm install
   npm run build
   ```
3. Chromeブラウザで `chrome://extensions/` にアクセスします。
4. 右上の「デベロッパーモード」をオンにします。
5. 「パッケージ化されていない拡張機能を読み込む」をクリックし、ビルドされた `dist` ディレクトリを選択します。

## 使い方と設定（Options）
拡張機能をインストール後、初めてポップアップを開くと自動的に**設定（Options）画面**が開きます。

### URLの許可設定
意図しないページでの誤作動を防ぐため、拡張機能を有効にしたいPukiWikiのURLを登録する必要があります。
設定画面には、ご自身のディレクトリのURLにワイルドカード（`*`）を付けた形式で入力してください。
例えば、ご自身のページが以下のようなURLである場合：
`https://example.com/pukiwiki/?Architecture-team/igarashi`

設定画面には、末尾にワイルドカード（`*`）を付けた以下の形式で入力してください。
```text
https://example.com/pukiwiki/?Architecture-team/igarashi*
```

**【スマートマッチ機能について】**
PukiWikiの編集画面は `?cmd=edit&page=...` のように間にパラメータが挿入されますが、本拡張機能はこれを内部で自動的に認識します。
また、新規作成時のようにURLが `%2F` でエンコードされていたり、末尾に `&refer=...` が付いている場合でも自動でデコード・マッチングを行います。
上記のように `?ディレクトリ名` の形式で設定しておけば、閲覧画面と編集画面、新規作成画面のいずれにも安全にマッチし、正常に動作します。

## 変換の仕様に関する注意点とTips
PukiWikiにはMarkdownと互換性のない独自の記法が多数存在します。
- PukiWiki独自の記法（例: `#contents`, `#article`, `#vote`, 複雑な `#ref` オプションなど）は、Markdown上でもそのままの文字列として維持（パススルー）されます。
- PukiWiki上で自動付与される見出しのアンカータグ（`[#x123456]`）は、編集の衝突を防ぐためMarkdown上でも維持されます。
- テーブルは PukiWikiの標準的な書式（`| xxx | yyy |`）をサポートしています。末尾の `h`（ヘッダ）、`f`（フッタ）、`c`（フォーマット）などのオプション指定がある場合は、内部的にコメントとして保持し、安全に双方向変換を行います。
- 画像の代替テキスト（`![alt](url)` の `alt` 部分）は、PukiWikiの `#ref` 仕様によるページ名解釈の誤動作（エラー）を防ぐため、反映時に安全のため自動的に省略され `&ref(url);` となります。

**💡 変換の仕様に関する注意点とTips:**
- **WikiLink (Obsidian記法) サポート:** Markdownエディタ内で `[[ページ名]]` と記述すると、Pukiwiki反映時に `[[ページ名>./ページ名]]` として自動変換されます。次にエディタを開いた際は `[ページ名](./ページ名)` として表示されるため、素早く内部リンクを作成できます。
- **深い箇条書きの制限保護:** PukiWikiは深さ3までの箇条書きしかサポートしていません。Markdown側で深さ4以上（例: `    - 4`）を入力してPukiwikiへ反映しようとした場合、データ消失を防ぐために自動的に深さ3へと調整されます。この際、確認のモーダルが表示されます。
- **非互換なHTMLタグの保護:** Markdown内に直接書かれた `<table>` などのHTMLタグはPukiWikiではサポートされていません。これらを検知した場合、反映前に警告モーダルを表示し、コメントアウトへのフォールバック変換を行います。
- **代替テキスト喪失の確認:** 前述の通り、Markdownの画像代替テキストはPukiWiki反映時に消去されます。そのため、代替テキスト付きの画像を挿入して反映しようとすると、「自動調整の確認」モーダルが表示され、テキストが欠落することを事前に確認できます。
- **ショートカットキー:**
  - `Cmd+Enter` (Mac) / `Ctrl+Enter` (Win) : 素早くPukiwikiへ反映（または確認モーダルを進める）
  - `Esc` : 各種モーダル（警告、Diff確認）をキャンセルしてエディタに戻る
  - `Cmd+/` : カーソル行のコメントアウト（PukiWikiの `//` との相互変換）
  - `Option+Enter` (Mac) / `Alt+Enter` (Win) : ドラフト（未保存の書きかけ）を素早く復元する

## 開発
- `src/converter.ts`: PukiWikiとMarkdownの双方向変換ロジック（正規表現ベース）
- `src/popup`: ポップアップUIとMarkdownエディタの制御
- `src/content_script.ts`: 実際のPukiWiki画面（DOM）とのテキスト受け渡し
- `src/background.ts`: URLフィルタリングとアクションの有効化/無効化制御
